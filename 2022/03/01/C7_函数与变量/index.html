一、函数

注意：

  1. C语言规定函数不能嵌套，只能多次调用
  2. 一个文件里由一个或者多个函数组成，这个文件就叫做”源程序文件”
  3. 大项目都是由多个源程序文件组成的，一般不会放在一个文件里面
  4. C程序从main函数开始执行，最终也是在main函数中结束整个程序的运行
  5. 一般不手动调用main函数，是留给系统调用的

函数的分类

  • 库函数：比如printf；直接使用，不需要我们自己定义
  • 自定义函数：自己定义的函数，目的是为了解决自己的需求

函数的定义和返回值

  • 函数参数：函数需要使用一些变量接受外部传入的数据，这些变量就是参数
  • 一般类型：返回类型 函数名(形参){执行语句+return返回值}
  • 注意：返回类型 void 表示函数没有返回值，void中return可以不写，也可以什么都不返回
  • 说明：
    1. 形参在函数调用之前并不分配内存，调用的时候才分配内存，调用结束后内存就被释放了，所以形参只能在函数内部使用！
    2. 实参可以是常量、变量和表达式，函数调用时，实参的值自动赋给形参，如果实参为数组名，那传递进去的就是数组首地址而不是变量
    3. 形参的数量和类型必须与实参的要保持一致
    4. 实参对形参是”值传递”，也就是单向传递，只能由实参传递给形参，不反过来传递
    5. C语言与其他语言不一样，可以有多条return语句，但return只能返回一个值
    6. 如果实际返回的类型与函数定义的返回类型不一致，则系统会自动转成函数本来返回值的类型

函数调用

  • 一般形式：函数名(实参列表);
  • 方式：
    1. 把函数作为一条语句调用
    2. 函数出现在表达式中，则称为函数表达式
    3. 函数调用作为函数的实参

函数声明(函数自己说明自己)

  • 一般形式：类型标识符 函数名(形参列表);
  • 注意：函数声明必须放在源代码的开头，才能保证正常的进行函数调用

函数可以嵌套调用，但是绝不能嵌套定义，在C语言中每一个函数都是平级关系！！！

函数递归调用

  • 调用栈：一块系统分配给我们这个程序的有特殊用途的内存，来保存形参、函数调用关系和局部变量，这个内存是有限的，超过内存大小时，就会崩溃
  • 递归中自己调用自己：
    • 注意：一般需要一个出口(递归结束条件)，否则就是死循环
    • 优点：代码简洁，精妙
    • 缺点：
      1. 不好理解
      2. 如果调用的层次太深，调用栈（内存）可能会溢出
      3. 效率和性能不高，深层次的调用保存的东西很多
      4. 有些问题，可能必须用递归解决（汉诺塔）
  • 递归函数的直接和间接调用
    • 直接调用：调用递归函数f的过程中，f函数又调用自己
    • 间接调用：调用f1的过程中要调用f2，然后f2函数过程中又要调用f1函数

数组元素作为函数参数：数组元素可以当作变量使用

数组名作为函数实参

  1. 传递的是数组首地址
  2. 如果实参传的是数组名，形参也应该是”数组名[]”或者”数组指针(09_C内容)”
  3. 这里是”地址传递”，而不是”值传递”(单向传递)，值传递形参是要分配内存的，实参与形参地址不一样；但是这里是把实参数组的开始地址传给了形参数组，形参与实参地址相同，这种传递叫”地址传递”
  4. 形参数组与实参数组类型要一致，比如都为int，否则会出错
  5. 形参数组大小可以不指定，即便指定了也可以实参数组大小不一致（C编译器对形参数组大小不做检查，只是将实参数组的首地址传递给形参数组）
  6. 形参的大小取决于实参的大小，实参具体是多大，形参赋值就只能改变多少值

多维数组名作为函数实参

  • 说明：
    • 是可以用多维数组名作为函数的形参和实参的
    • 形参数组在定义时，可以指定每一维的大小，也可以省略第一维大小，但不能省略第二维大小
  • 技巧：形参定义尽量和实参一样几行几列，就不容易出错

二、变量

局部变量

  • 简介：在一个函数内部定义的变量，只在本函数范围内有效，其他函数无法使用
  • 说明：
    1. 不同的函数可以使用相同的变量名，互不干扰
    2. 形参也是个局部变量，只在自己函数里有效，其他函数不能调用
    3. 特殊写法：直接写一个{}代码块，里面定义的变量也是局部变量

全局变量

  • 简介：在函数外部定义的变量就是全局变量，可供多个函数共用
  • 说明：范围是从定义开始位置到本源程序文件结束
  • 总结：在一个函数中，既可以使用本函数中的局部变量，又可以使用有效的全局变量
  • 优点：增加了函数与函数之间的数据联系渠道，
  • 在各个函数之间有了直接的传递通道，不再需要通过实参形参传递参数值了
  • 缺点：
    1. 有必要再使用，谨慎使用，因为整个程序运行周期之间都会占内存
    2. 降低了函数的通用性，因为函数执行时要依赖这些外部的全局变量了，不方便迁移
    3. 全局变量多了，降低了程序的清晰性和可读性，因为到处都可以改变全局变量的值

外部变量说明：

  • 一般形式：extern 类型标识符 变量名1,变量名2;（例：extern int a,b;）
  • 作用：
    1. 在include了其他文件时，想在本文件使用其他文件的全局变量时，就用extern说明这个变量是外部文件的变量
    2. 后面定义的全局变量，前面的函数也想使用，可以在文件头部进行外部变量说明达到使用的目的
  • 说明：
    1. 不分配内存，相当于函数声明
    2. 可以进行多次变量声明，也可以在函数内部声明，那么只在函数内部有效

同一个文件中，如果全局变量和局部变量同名，则在局部变量作用范围内，全局变量不起作用

变量的存储类别

  • 静态存储变量：在程序运行期间分配固定存储空间的变量
  • 动态存储变量：在程序运行期间根据需要进行动态分配存储空间的变量
  • 说明：
    1. 全局变量是静态存储变量
    2. 动态存储变量包括局部变量、函数形参和函数调用时现场的一些数据和返回地址等；这些数据再函数开始时分配存储空间，函数调用完毕，这些空间就被释放掉了

局部静态变量：

  • 简介：局部变量的存储方式一般为动态分配存储空间，但也可以静态储存
  • 一般形式：用static加以说明，占用的存储单元不释放，下一次调用该函数时，变量的值是上一次结束时的值（例：static int a = 10;）
  • 说明：
    1. 在静态存储区内分配存储单元，程序整个运行期间不释放，初值只赋予一次
    2. 定义静态局部变量时，如果不赋初值，则系统自动赋初值为零；而正常局部变量不赋初值就是一个随机未知的值
    3. 静态局部变量其他函数也不能使用
  • 缺点：长期占用内存，降低程序可读性，除非必要尽量不用

三、全局变量跨文件引用

  • 方法：在引用该全局变量的文件的头部使用extern做外部变量说明
  • 注意：在定义全局变量时前面增加static，则该全局变量只能在定义它的文件中使用，不能跨文件

四、函数的跨文件调用（内部函数和外部函数）

  • 一般形式：static 类型标识符 函数名(形参表){…}
  • 说明：
    • 内部函数(静态函数)只能被本文件中其他函数调用
    • 不同文件内的静态函数名可以一样，不受影响，因为都只能在自己所在的文件中使用
  • 外部函数：不用static的函数就是外部函数，默认就有(extern)，不用自己加
  • 跨文件调用方法：只需要在需要调用的文件前面进行函数声明即可，要不要extern都可以

五、static关键字用法总结

  1. 函数内部定义变量时使用，则该变量会保存在静态存储区，不多次初始化且每次调用继承上一次的值
  2. 全局变量之前加static时，则该全局变量只能在本文件中使用，不能跨文件使用
  3. 函数定义之前加static时，则该函数只能在本文件中被调用，也不能跨文件调用
