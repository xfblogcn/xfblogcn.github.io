一、正则表达式

什么是正则表达式？

正则表达式，即一个定义了匹配规则的特殊字符串，该字符串中包含了一些特定字符，其中普通字符用于精确匹配，特定字符具有明确的匹配规则。如果某个字符串符合正则表达式定义的所有匹配规则，那么该字符串就与该正则表达式匹配（在线测试工具）

  • 模块：标准库模块re是专门用于支持正则表达式的

  • 作用：正则表达式是字符串处理的利器，它可以用于字符串的匹配、查找、替换和劈分

  • 注意：正则表达式在很多编程语言中都是适用的，且绝大部分语法都是通用的

1. 原始字符串

  • 原始字符串：是一种字符串字面值的表示方式，通常用于处理正则表达式模式、文件路径或其他需要特殊字符的情况（其中特殊字符不会被转义，而是将其解释为普通字符）
  • 解释：
    • 在原始字符串中，包含了哪些字符就表示哪些字符，而无需考虑转义字符
    • 如果使用普通字符串来表示正则表达式，则需要考虑转义字符
  • 重要概念：
    • 转义字符：简单来说就是在一些字符前加上反斜杠 \ 使它具有其他意义，例\n、\\、\r、\t、\"等
    • 字符的字面值：简单的说，就是要确切的显示出一个字符，即一个没有任何意义的字符
    • 正则表达式中具有特殊含义的字符：例\ . ^ $ ? + * {} [] () |，想要使用这些字符的字面值，就必须使用反斜杠进行转义

2. match方法匹配（图）

  • match(pattern, string[, flags])：

    • 作用：该方法会根据参数string指定的字符串与参数pattern指定的正则表达式进行匹配

    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象

    • 参数string：需要匹配的普通字符串

    • 参数flags：是一个标志位，用于控制正则表达式的匹配方式。如果需要同时使用多个，可以使用 | 对标志位进行分隔

  • 匹配方式：该方法会从参数string指定的字符串的开头开始，一直向后尝试匹配参数pattern指定的正则表达式，在到达pattern的末尾前，如果遇到无法匹配的字符或到达了string的末尾，都表示匹配失败。反之，如果到达pattern的末尾时，所有的字符都是匹配成功的，则表示匹配成功，且终止匹配，不再对string向后匹配

  • 返回值：若匹配成功，则返回一个Match对象；若匹配失败，则返回None

3. 正则表达式的预编译

  • compile(pattern[, flags])：

    • 作用：该方法用于预编译正则表达式，使得在重复使用一个正则表达式时不用多次编译，以提高效率

    • 参数pattern：是一个正则表达式

    • 参数flags：是一个标志位，与方法match()中参数flags的含义完全相同

  • 返回值：该方法会返回对正则表达式预编译之后得到的对象

  • 知识点：

    • 预编译之后得到的对象提供了方法 match(string[, pos[, endpos]]) ，即对应模块re中的match方法

    • 参数pos：用于指定被匹配的字符串的起始位置，默认值为0

    • 参数endpos：用于指定被匹配的字符串的结束位置，默认值是字符串的长度（左闭右开）

4. 单个字符的匹配（图）

注意：正则表达式里的中括号[ ]具有去除字符的特殊含义的作用，从而只剩下字符的字面值

  • 中括号 [] 的使用：
    • 当 ^ 是中括号内的第一个字符时，表示否定
    • 当 - 不是中括号内的第一个字符时，表示范围
    • 在中括号中可以使用运算符&，表示并且

5. 贪婪与勉强模式（图）

. ：点号是通配符，表示匹配任何单个字符，除了换行符 \n 之外的任何字符

* ：星号是量词，表示匹配前面的元素零次或多次

? ：问号也是量词，表示匹配前面的元素零次或一次

  • 贪婪模式：
    • 在贪婪模式下，正则表达式引擎会尽可能多地匹配符合模式的文本（它会匹配尽量长的字符串）
    • 贪婪模式使用 * 和 + 量词（**.* 表示匹配任意多个字符，直到不能再匹配为止**）
  • 勉强模式（非贪婪模式）：
    • 在勉强模式下，正则表达式引擎会尽可能少地匹配符合模式的文本（它会匹配尽量短的字符串）
    • 勉强模式使用 *? 和 +? 量词（**.*? 表示匹配任意多个字符，但尽可能少**）

6. 边界匹配（图）

  • 边界匹配：是用于确定匹配的文本边界的一种技术，主要用于匹配字符串的边界或单词的边界
  • 注意：\b和\B是单词边界，只用于识别边界，而不匹配任何实际字符，所以是看不到的
    • \b：表示字母数字与非字母数字的边界， 非字母数字与字母数字的边界
    • \B：表示字母数字与非非字母数字的边界，非字母数字与非字母数字的边界

7. 分组匹配与逻辑匹配

逻辑匹配：即使用竖线 | 来表示逻辑或操作

  • 如果 | 被使用在分组中，则作用范围是那个组()内

  • 如果 | 没有被使用在()中，则作用范围是整个正则表达式

  • 分组（group）：是一种将模式中的子表达式包装在括号内的机制。分组允许对匹配的文本进行更详细的控制，例如捕获特定部分的文本，对不同部分应用不同的量词，或者引用分组中的文本
  • 捕获分组：使用小括号 ( ) 来创建捕获分组，以捕获匹配的文本，捕获分组可以在匹配后被引用
    • 捕获分组通常通过数字编号来引用，语法格式为：\编号
  • 非捕获分组：使用 (?: ) 创建非捕获分组，不会捕获匹配的文本，只用于分组操作，不会在匹配后被引用
    • (?:abc)+ 表示匹配一个或多个 “abc”，但不会捕获这些文本
  • 命名分组：使用 (?P<name> ) 创建命名分组，以便在匹配后使用名称引用分组中的文本
    • 命名分组通常通过别名来引用，语法格式为：**?P=别名**

三、正则常用方法

1. search方法搜索

  • 方法：search(pattern, string[, flags])，该方法会在参数string指定的字符串中查找参数pattern指定的第一个字符串

    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象
    • 参数string：需要查找的普通字符串

  • 返回值：如果有匹配的子串，则返回第一个匹配对应的Match对象；如果没有匹配的子串，则返回None

  • 注意：除了直接调用模块re的方法search()之外，也可以调用模块re的方法compile()的返回值的方法search(string[, pos[, endpos]])

    • 参数pos用于指定被查找的字符串的起始位置，默认值为0
    • 参数endpos用于指定被查找的字符串的结束位置，默认值是字符串的长度。被查找的子串不包括结束位置，类似左闭右开
    • 模块re的方法search()中的参数pattern和flags被转移到了方法compile()中

python正则表达式函数match()和search()的区别：

  • match()函数是从字符串的开头进行匹配，只有在字符串开头匹配到模式时才会返回匹配结果。如果模式在字符串开头没有匹配到，则match()函数返回None
  • search()函数则是在整个字符串中搜索模式，只要找到第一个匹配的结果，就会返回匹配结果。如果模式在整个字符串中没有匹配到，则search()函数返回None

2. findall方法查找

  • 方法：findall(pattern, string[, flags])，该方法会在参数string指定的字符串中查找参数pattern指定的所有子串

    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象
    • 参数string：需要查找的普通字符串

  • 返回值：如果有匹配的子串，则返回所有匹配的子串组成的列表；如果没有匹配的子串，则返回空列表

  • 注意：除了直接调用模块re的方法findall()之外，也可以调用模块re的方法compile()的返回值的方法findall(string[, pos[, endpos]])

    • 参数pos用于指定被查找的字符串的起始位置，默认值为0
    • 参数endpos用于指定被查找的字符串的结束位置，默认值是字符串的长度。被查找的子串不包括结束位置，类似左闭右开
    • 模块re的方法findall()中的参数pattern和flags被转移到了方法compile()中

3. finditer方法查找

  • 方法：finditer(pattern, string[, flags])，该方法会在参数string指定的字符串中查找参数pattern指定的所有子串

    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象
    • 参数string：需要查找的普通字符串

  • 返回值：如果有匹配的子串，则返回所有匹配的子串组成的迭代器；如果没有匹配的子串，则返回不包含任何元素的迭代器

  • 注意：除了直接调用模块re的方法finditer()之外，也可以调用模块re的方法compile()的返回值的方法finditer(string[, pos[, endpos]])

    • 参数pos用于指定被查找的字符串的起始位置，默认值为0
    • 参数endpos用于指定被查找的字符串的结束位置，默认值是字符串的长度。被查找的子串不包括结束位置，类似左闭右开
    • 模块re的方法finditer()中的参数pattern和flags被转移到了方法compile()中

4. sub方法替换

  • 方法：sub(pattern, repl, string[, count][, flags])，该方法会将参数string指定的字符串中所匹配参数pattern的子串替换为参数repl指定的字符串

    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象
    • 参数string：需要查找的普通字符串
    • 参数repl：既可以指定为替换的字符串，还可以指定为一个函数。通过该函数，可以把每次匹配对应的Match对象作为函数的输入参数，在函数体中对匹配的子串进行处理，函数的输出返回值作为替换的字符串
    • 参数count：用于指定最大替换次数，默认值为0，表示替换所有的匹配

  • 返回值：替换成功后得到的字符串

  • 应用：sub()方法也可以实现字符串中子串的删除，将repl参数（替换字符串）设置为空字符串即可

  • 方法：subn()(pattern, repl, string[, count][, flags])，其用法与sub()方法完全一致，不同的是返回值为包含两个元素的元组，即对应的方法sub()的返回值、替换次数

  • 注意：除了直接调用模块re的方法sub()和subn()之外，也可以调用模块re的方法compile()的返回值的方法**sub(repl, string[, count])、subn(repl, string[, count])**。模块re的方法sub()和subn()中的参数pattern和flags被转移到了方法compile()中

5. split方法劈分

  • 方法：split(pattern, string[, maxsplit][, flags])，该方法会根据参数pattern指定的劈分符对参数string指定的字符串进行劈分
    • 参数pattern：是一个正则表达式，或对正则表达式预编译之后得到的对象
    • 参数string：需要劈分的普通字符串
    • 参数maxsplit：用于指定最大劈分次数，默认值为0，表示不限制劈分次数
  • 返回值：劈分后所有子串组成的列表
  • 注意：除了直接调用模块re的方法split()之外，也可以调用模块re的方法compile()的返回值的方法**split(string[, maxsplit])**。模块re的方法split()中的参数pattern和flags被转移到了方法compile()中

6. Match对象属性和方法（图）

  • 简介：模块re的方法match()和search()在匹配成功时都会返回一个Match对象，该对象包含了很多关于此次匹配的信息，可以通过访问Match对象提供的属性和方法来获取这些匹配信息
