二、数据类型

重要概念：

  • 字节：一字节(Byte)等于八位(bit)。例如int的存储宽度是4个Bytes，即32个bit，存储范围到2**32
  • 字符：一个英文字符等于一个字节，一个中文字符等于三个字节（utf8字符集）
  • 数据类型后指定的宽度：此指定的宽度为字符（除整型外），而不是字节！

1. 数值类型

  • 关于整数：整数类型int()的显示宽度默认值为存储范围的最大值的宽度上加1位，因为数值类型默认都是有符号的，符号位也占1宽度。为整数类型指定宽度时，仅仅只是指定查询结果的显示宽度，与存储范围无关，因为整数类型的存储宽度是固定的

  • 关于浮点数：在浮点类型中，对于float[(m, d)]、double[(m, d)]和decimal[(m, d)]，m表示的是数字总个数，d表示的是小数点后个数。其中，float和double的m最大值为255，decimal的m最大值为65，但它们的d最大值都为30

2. 日期类型

  • 字段分类：

    • year：YYYY（1901/2155）
    • date：YYYY-MM-DD（1000-01-01/9999-12-31）
    • time：HH:MM:SS（’-838:59:59’/‘838:59:59’）
    • datetime：YYYY-MM-DD HH:MM:SS（1000-01-01 00:00:00/9999-12-31 23:59:59）
    • timestamp：YYYYMMDD HHMMSS（1970-01-01 00:00:00/2037 年某时）

  • datetime与timestamp的区别：

    1. datetime的日期范围是1001/9999年，timestamp的时间范围是1970/2038年
    2. datetime存储时间与时区无关，timestamp存储时间与时区有关，显示的值也依赖于时区。在mysql服务器、操作系统以及客户端连接都有时区的设置
    3. datetime使用8字节的存储空间，timestamp的存储空间为4字节。因此，timestamp比datetime的空间利用率更高
    4. datetime的默认值为null，timestamp的默认值为当前时间current_timestamp，如果不做特殊处理，并且update语句中没有指定该列的更新值，则默认更新为当前时间

  • 注意：针对datetime和timestamp如果是用作注册时间，那么前者指定not null default now()，后者不指定任何值，即可自动填充时间；如果是用作更新时间那么需要额外指定on update now()，该配置timestamp自带，所以也只需datetime指定（mysql版本不同会导致explicit_defaults_for_timestamp值不同，只有当其值为off时，以上结论才有效）

    • 查询当前explicit_defaults_for_timestamp值：show [global] variables like '%timestamp%';
    • 设置当前explicit_defaults_for_timestamp值：set [global] explicit_defaults_for_timestamp=off;

3. 字符类型

char和varchar指定的参数都是字符的长度，而不是字节！（一个英文字符占1个字节，一个中文字符占3个字节）

  1. length()：查看字节数
  2. char_length()：查看字符数

  • char类型：定长，简易，浪费空间，存取速度快，存取字符长度0-255

    • 存储：存储char类型的值时，会往右填充空格来补足长度
    • 检索：在检索（查询）时，查询出的结果会自动删除尾部的空格（可以通过设置set sql_mode='PAD_CHAR_TO_FULL_LENGTH';以查询char_length()正常值）

  • varchar类型：变长，精准，节省空间，存取速度慢，存取字符长度0-65535

    • 存储：存储varchar类型的值时，不会用空格填充，会精准存储，字符串是什么就存什么
    • 检索：在检索（查询）时，也会精准查询，包括字符串尾部的空格也会保存下来

  • char和varchar存储方式的区别（重点）：

    • varchar类型会在真实数据前加1-2bytes的前缀，该前缀用来表示真实数据的bytes字节数（1-2bytes最大表示65535个数字，正好符合mysql对row的最大字节限制，即已经足够使用）

      Value char(4) Storage Required varchar(4) Storage Required
      ‘’ ‘ ‘ 4 bytes ‘’ 1 bytes
      ‘ab’ ‘ab ‘ 4 bytes ‘ab’ 3 bytes
      ‘abcd’ ‘abcd’ 4 bytes ‘abcd’ 5 bytes
      ‘abcdefg’ ‘abcd’ 4 bytes ‘abcd’ 5 bytes

  • 重点：即使char和varchar的存储方式不太相同，但是对于两个字符串的比较，都只比较其值，会忽略char值存在的右填充（即使设置了set sql_mode='PAD_CHAR_TO_FULL_LENGTH';也一样）。但这不适用于模糊查询like，like不会仅比较其值，还包括其末尾空格等

建议与技巧：

  1. 在不缺存取空间的今天，我们更应该使用char类型存储，以提高存取性能。但对于不要求存取性能的数据，更应该使用varchar类型存储，例如存入过后长时间不需要使用的数据，使用varchar更能节省空间
  2. 当我们创建表时，应该将char类型数据创建于varchar类型数据之前，且在同一张表中，尽量不要char和varchar混合用，能只用一种就只用一种
  3. 在数据库中，不建议存储容量特别大的文件，所以其余字符串类型也很少使用，效率也不高：char>varchar>text
  4. 对于innodb存储引擎，建议使用varchar类型。因为对于innodb数据表，内部的行存储格式没有区分固定长度和可变长度列（所有数据行都使用指向数据列值的头指针），因此在本质上，使用固定长度的char列不一定比使用可变长度varchar列性能要好。所以，主要的性能因素是数据行使用的存储总量。由于char平均占用的空间多于varchar，因此使用varchar来最小化需要处理的数据行的存储总量和磁盘I/O是比较好的。

4. 枚举类型与集合类型

  • enum：单选，只能在给定范围内选一个值，若不是给定范围内的值则变成空值
  • set：多选，在给定范围内可以选择一个或一个以上的值

三、字段约束类型

  • not null与default：一般配合使用，即not null default 'xxx'表示不传值或传空值则使用默认值
  • **unique [key]**：
    • 单列唯一：限制单个字段唯一。创建表时可以直接在字段后添加约束条件，也可以在字段创建完后使用多个unique(字段名)声明单列唯一
    • 联合唯一：限制多个字段唯一，即两个字段值的组合要保证唯一。创建表时不在字段后跟约束条件，而是在字段创建完后使用一个unique(字段1,字段2,...)声明联合唯一，此时字段1的key在数据库中为MUL
  • primary key：主键，限制不为空且唯一（not null unique）。对于innodb存储引擎，一张表必须要有一个主键，如果没有设置主键，则将表中不为空且唯一的字段作为主键，如果没有这样的字段，则将一个隐藏的字段作为主键，只是无意义
    • 单列主键：创建表时可以直接在字段后添加约束条件，也可以在字段创建完后使用parmary key(字段名)声明单列主键
    • 复合主键：创建表时不在字段后跟约束条件，而是在字段创建完后使用parmary key(字段1,字段2,...)声明复合主键，此时约束即为联合唯一
  • auto_increment：自增长的字段必须是一个key。可以自己指定自增长字段的值，则下次自增长从指定的值开始。
    • 查看auto_increment相关的变量：show variables like 'auto_inc%'
    • 步长：auto_increment_increment
    • 起始偏移量：auto_increment_offset（起始偏移量的设置一定要小于等于步长，因为第一步不能迈过步长）

重点：删除表后auto_increment值并未重置？清空表

  • 问题：对于使用了auto_increment字段的表，如果用delete from tbname清空表，并不会将表中auto_increment的值重置为1（可通过show create table tbname查看），所以清空表最好不用delete语句，因为delete语句一般搭配where使用
  • 解决：使用truncate tbname清空表，可以将表中auto_increment的值重置为1
  • foreign key：外键，用于建立表之间的关系。使用时，将主表的字段通过约束条件foreign key(字段) references tb2(tb2的字段)与被关联表的字段进行绑定（必须保证被关联的字段唯一才能关联）。
    • 插入数据时，先往被关联表插数据，主表才能插入数据成功
    • 删除数据时，先删除主表的数据，才能删除被关联表对应的数据。而这样删除太繁琐，可以在建立外键约束条件时加入条件on delete cascade on update cascade，即删除同步与更新同步，这样当删除被关联表数据时，主表的对应数据会被自动同步删除

四、数据库/表/记录操作

SQL语言主要用于存储数据、查询数据、更新数据和管理关系数据库系统

SQL语言由IBM开发，分为三类：

  1. DDL语句 - 数据库定义语言：数据库、表、视图、索引、存储过程，例如CREATE、DROP、ALTER等
  2. DML语句 - 数据库操纵语句：插入数据INSERT、删除数据DELETE、更新数据UPDATE、查询数据SELECT
  3. DCL语句 - 数据库控制语言：例如控制用户的访问权限GRANT、REVOKE等

1. 数据库（database）

  • 系统数据库（不允许操作）：
    • information_schema：虚拟库，不占用磁盘空间，存储的是数据库启动后的一些参数，如用户表信息、列信息、权限信息、字符信息等
    • mysql：授权库，主要存储系统用户的权限信息
    • performance_schema：主要用于收集数据库服务器性能参数，记录处理查询请求时发生的各种事件、锁等现象
    • sys：通过这个库可以快速的了解系统的元数据信息
  • 数据库命名规则：

    • 唯一性：在同一MySQL服务器上，每个数据库的名称必须是唯一的，不能重复

    • 组成：数字、字母（区分大小写）和下划线，必须以字母开头，不能以数字或下划线开头

    • 保留字：避免使用MySQL的保留字（例SELECT、INSERT等）作为数据库名，以防止与SQL查询语句产生冲突

  • 数据库使用注意：
    1. 对数据库进行任何操作，应首先指定数据库：use 数据库名;
    2. 在mysql中正确的设置字符集，应该是utf8，而不是utf-8！

2. 数据库操作

  • 增：create database 数据库名 charset [=] utf8;
  • 删：drop database 数据库名;
  • 改：alter database 数据库名 charset utf8;（只能修改字符集，不能修改数据库名）
  • 查：
    • show databases;（查看所有数据库）
    • show create database 数据库名;
    • select database();（查看当前正在使用的数据库）

3. 数据表（table）

  • 创建表的注意事项：
    1. 在同一张表中，字段名不能相同
    2. 宽度和约束条件是可选的
    3. 字段名和类型是必选的
    4. 表中的最后一个字段不要加逗号
  • 数据表之间的关系：
    • 多对一：表1的多条记录能对应表2的一条记录，但表2的多条记录不能对应表1的一条记录
    • 外键设置：在表1下对需要关联的字段设置外键 foreign key(tb2_id) references tb2(id)
    • 多对多：表1的多条记录能对应表2的一条记录，表2的多条记录也能对应表1的一条记录
    • 外键设置：此时两张表都需要设置外键，那么先建哪张表都不对，而是再创建第三张表专门用于存储这两张表的关系，并且在 tb3 中分别对 tb1_id 和 tb2_id 设置外键 tb1(id) 和 tb2(id)
    • 一对一：表1的一条记录只能对应表2的一条记录，表2的一条记录也只能对应表1的一条记录
    • 设置外键：按照逻辑分析出对应关系，在其中一张表中建立外键，但由于外键是可以重复的，所以外键本身就是多对一的关系，所以在创建外键字段时应该添加约束条件唯一（unique）

4. 数据表操作

  • 增：cerate table 表名(字段名 类型[(宽度) 约束条件],...);

  • 删：drop table 表名;

  • 改：

    • alter table 表名 rename 新表名;
    • alter table 表名 add 字段名 数据类型 [完整性约束条件...] frist;
    • alter table 表名 add 字段名 数据类型 [完整性约束条件...] after 字段名;
    • alter table 表名 drop 字段名;
    • alter table 表名 modify 字段名 数据类型 [完整性约束条件...];
    • alter table 表名 change 旧字段名 新字段名 新/旧数据类型 [完整性约束条件...];

  • 查：

    • desc 表名;（查看表结构）
    • show create table 表名\G;（查看表详细结构，可加\G）

  • 复制表：

    • 复制表结构+记录，key（主键、外键和索引）不会复制：create table new_tb1 selete * from tb1;
    • 只复制表结构，使用like：**create table new_tb1 like tb1;**
    • 只复制表结构：使用where限制数据：create table new_tb1 selete * from tb1 where 1>2;

5. 数据记录操作

注意：主键列id是自动增长，但是在全列插入时必须需要占位，通常使用0、default或null来占位，插入成功后以实际数据为准

  • 增：

    • 插入一条记录：insert into 表名[(字段1,字段2,...,字段n)] values(值1,值2,...,值n);

    • 插入多条记录：insert into 表名 values(值1,值2,...,值n),(值1,值2,...,值n),...;

    • 插入查询结果：insert into 表名(字段1,字段2...) select (字段1,字段2...) from 表2 where ...;

  • 删：

    • 物理删除：delete from 表名 where 条件;
    • 逻辑删除：update 表 set is_delete=1 where 条件
    • 注意：逻辑删除本质是修改is_delete字段操作，然后通过python代码处理后隐藏

  • 改：update 表名 set 字段1=值1,字段2=值2,...,字段n=值n where 条件;

  • 查：

    • 查询所有数据：select \* from 表名
    • 查询完整语法：select distinct 字段 from 表 where 条件 group by 分组条件 having 过滤语句 order by 排序字段 limit n;

五、单表查询

查询时定义显示格式（字符串函数）：

  • concat( )：函数用于连接字符串，和python打印一样，实参之间用逗号分隔
  • concat_ws( )：函数用于连接字符串，第一个参数为分隔符，实参之间用逗号分隔

1. where过滤

  • 语法：select * from 表名 where 条件;

  • 比较运算符：> < = >= <= !=或<>

  • 逻辑运算符：and、or、not

  • 模糊查询like：%表示任意多个任意字符，_表示一个任意字符

  • 范围查询：

    • in 表示在一个非连续的范围内（成员运算符）
    • between … and … 表示在一个连续的范围内，闭区间（左闭右闭）

  • 空判断：is null、is not null（null与’’是不同的）

优先级：小括号 > not > 比较运算符 > 逻辑运算符

注意：and比or先运算，如果同时出现并希望先算or，需要结合()使用

2. order by排序

  • 语法：select * from 表名 order by 字段1 asc|desc [,字段2 asc|desc ,…];
  • 解释：将数据按照字段1进行排序，如果某些字段1的值相同时，则按照字段2排序，以此类推
  • asc：从小到大排序，即升序（mysql默认是从小到大排序的）
  • desc：从大到小排序，即降序
  • 组合排序：order by 字段1 desc,字段2 asc，即先按照字段1降序排，当字段1相同时，按照字段2升序排

3. 聚合函数

  • 总数：count(*)表示计算总行数，括号中写星号或列名，结果是相同的
  • 最大数：max(列)表示求此列的最大值
  • 最小值：min(列)表示求此列的最小值
  • 求和：sum(列)表示求此列的和
  • 平均值：avg(列)表示求此列的平均值

4. group by分组

  • 含义：将查询结果按照1个或多个字段进行分组，字段值相同的为一组
  • 用法：group by一般不单独使用，单独对于某个字段使用类似于distinct去重，一般配合group_concat()使用
  • **group_concat()**：表示分组之后，根据分组结果来放置每一组的某字段的值的集合，作为输出字段来使用
  • group by + 聚合函数：聚合函数是对分组后的每一组的内容计算，若是没有分组，则默认为一组
  • group by + having：having用来分组查询后指定一些条件来查询结果，其作用和where一样，但having只能用于group by
  • group by + with rollup：with rollup的作用是在最后新增一行，来记录当前列里所有记录的总和

having过滤与where约束的区别：

  1. 执行优先级从高到低：where > group by > having > select
    • where 子句在数据从表中检索之前进行过滤，它用于筛选行
    • group by 子句用于分组数据
    • having 子句在分组之后对分组数据进行过滤
    • select 子句用于选择要包含在结果集中的列
  2. where发生在分组group by之前，可以包含任意字段，通常用于对原始数据进行过滤，而不涉及聚合函数
  3. having发生在分组group by之后，只有分组后的字段和聚合函数可以在having中使用，并且可以包含聚合函数

5. limit分页

  • 语法：select * from 表名 limit start,offset;

  • limit offset == limit 0,offset：默认初始位置为0（可省略），然后往后查几条（offset）

  • limit start,offset：从第几条开始（start），即先查询出第几条，然后包含这一条在内往后查几条（offset）

  • *注意：start依然是页数的意思，只是需要通过计算得出start的值，计算公式为 (第N页-1)*每页的个数*

七、连接查询

  • 内连接查询（inner join…on…）：

    • 作用：用于返回两个表中匹配的数据，类似于两个表的交集，仅返回同时在两个表中存在的行
    • 语法：select * from 表1 inner join 表2 on 表1.列 = 表2.列;

  • 左连接查询（left join…on…）：

    • 作用：用于返回左表的所有行和右表中匹配的行，如果右表没有匹配行，将为右表的那部分返回 NULL 值
    • 语法：select * from 表1 left join 表2 on 表1.列 = 表2.列;

  • 右连接查询（right join…on…）：

    • 作用：用于返回右表的所有行和左表中匹配的行，如果左表没有匹配行，将为左表的那部分返回 NULL 值
    • 语法：select * from 表1 right join 表2 on 表1.列 = 表2.列;

  • 用法：inner join用于获取精确匹配数据，而left join和right join用于保留一个表（基准表）的所有数据，同时合并另一个表的匹配数据（不管什么连接查询，都可以使用as给表取别名）

  • 注意：左连接查询完全可以实现右连接查询的功能，所以右连接查询一般不常用

八、自关联查询

  • 简介：自关联是一种在同一表内连接不同行的技术，通常用于处理具有父子关系或层级关系的数据。在这种情况下，表中的每一行可以与表中的其他行建立关联，以形成层次结构

九、子查询

  • 概念：子查询是一种在SQL查询中嵌套的查询，它可以用于从一个查询中获取数据，并将其作为另一个查询的一部分；通常嵌套在select、from、where、having或in等子句中，以获取或筛选数据

  • 注意：数据量过大时，使用子查询可能会导致性能问题（使用时最好用括号括起来，表示为条件）

  • 标量子查询：返回单个值，通常用于比较运算或在主查询中的某个列中设置值

  • 列级子查询：返回一个列，通常用于选择主查询中的一个或多个列的值

  • 行级子查询：返回一行数据，通常用于与主查询中的一行或多行进行比较或联接

十、MySQL视图

  • 简介：MySQL视图（View）是一种虚拟的数据库对象，它是一个由一个或多个基本表（或其他视图）的查询结果组成的结果集，可以像表一样被查询

  • 创建视图：create view v_视图名称 as select语句;（视图名称一般以v_开头命名）

  • 查看视图：show tables;（查看表会将所有的视图也列出来）

  • 使用视图：视图的用途就是查询，可以像查询普通表一样使用 select 语句来查询视图

  • 删除视图：drop view 视图名称;

  • 视图的作用：

    1. 视图常用于对select语句封装，类似python中的函数，提高了查询语句重用性
    2. 视图可以用来隐藏底层表的复杂性，提供一个更简单和安全的数据访问层
    3. 视图可以用于限制用户对数据的访问权限，只允许他们看到特定列或行
    4. 视图还可以用于在多个应用程序组件之间共享数据，提供一种抽象的数据接口

  • 注意：

    • 视图是虚拟的，查询视图时实际上是在运行视图定义的查询，因此可能会引入一些性能开销
    • 对于频繁使用的查询，考虑将它们转化为物理表以提高性能

十一、事务

  • 事务的特性：

    • 原子性：事务是一个不可分割的工作单元，要么全部执行成功，要么全部回滚
    • 一致性：事务在执行前后，数据库必须保持一致性状态。如果事务开始时数据库是一致的，那么在事务结束后，数据库仍然必须是一致的
    • 隔离性：多个事务同时执行时，它们应该被隔离开来，一个事务的修改不应影响其他事务的执行。MySQL支持不同级别的隔离度，如读未提交、读提交、可重复读和串行化
    • 持久性：一旦事务成功提交，其结果应该持久保存在数据库中，即使发生了系统故障也不应丢失

  • 事务的控制语句：

    • begin 或 start transaction：开始一个新的事务
    • commit：提交事务，将所有已执行的操作永久保存到数据库中
    • rollback：回滚事务，撤销所有已执行的操作
    • savepoint：设置保存点，用于在事务中部分回滚
    • rollback to：回滚到指定的保存点

  • 事务的自动提交：默认情况下，MySQL是以自动提交模式运行的，这意味着每个SQL语句都被视为一个单独的事务，并且在执行后立即提交（使用 set autocommit = 0; 可以禁用自动提交）

十三、索引

  • 简介：索引是一种数据结构，它存储了表中一列或多列的值以加速数据的检索，这提供了一种更快速地查找数据行的方法，而不必扫描整个表
  • 作用：
    1. 提高查询性能：索引可以加速查询操作，使其更快
    2. 唯一性约束：索引可以确保表中某一列的值是唯一的
    3. 外键关联：索引可以用于实现表之间的外键关联
    4. 排序：索引可用于加速排序操作
  • 索引分类：
    • 主键索引（Primary Key Index）：用于唯一标识每一行的索引
    • 唯一索引（Unique Index）：确保索引列中的值是唯一的
    • 普通索引（Normal Index）：最基本的索引类型，用于加速查询
    • 全文索引（Full-Text Index）：用于全文搜索操作
    • 组合索引（Composite Index）：多个列上的索引，用于加速复合条件查询
    • 空间索引（Spatial Index）：用于空间数据类型（如地理信息数据）的查询
    • 全局索引（Global Index）和局部索引（Local Index）：针对分区表的索
