一、线程

重要概念：

  • 进程：是系统进行资源分配和调度的一个独立单位。即运行中的应用程序，应用程序是静态的，而进程是动态的
  • 线程：是操作系统能够进行运算调度的最小单位。它被包含在进程之中，是进程中的实际运作单位，一个进程中可以并发多个线程，每条线程并行执行不同的任务
  • 并发：多个任务交替执行的方式称为并发
  • 并行：多个任务同时执行的方式称为并行（不管是单核 cpu 还是多核 cpu 都支持多任务）

进程与线程的关联：

  1. 每个进程至少对应一个线程。每个进程用来完成一件任务，每个线程用来完成进程内的一件子任务
  2. 每个进程都有独立的内存空间，而进程内的所有线程共享内存空间

1. 自动创建与启动的线程

  • 简介：任何进程都会自动创建并启动一个线程，该线程被称为父/主线程，其默认名称为 MainThread

2. 子线程的创建1

  • 使用类对象 Thread 创建并启动子线程：标准库模块 threading 中提供了一个类对象 Thread，用于表示线程

  • 应用：根据类对象 Thread 创建线程实例对象，调用线程实例对象的方法 start() 即可启动线程（调用方法 start() 后，先会自动调用 Thread 类中的 run() 方法，方法run()又会自动调用参数 target 指定的函数）

  • 创建线程实例对象时必须指定关键字实参：

    • 其构造方法定义为：__init__(self, group=None, target=None, name=None, args=(), kwargs=None, *, daemon=None)
    1. 参数 group：用于指定线程实例对象所属的线程组，默认不属于任何线程组
    2. 参数 name：用于指定创建的线程实例对象的名称，第n个子线程的默认名称为’Thread-n’
    3. 参数 target：用于指定被方法run()调用的函数，默认没有函数被调用
    4. 参数 args：用于指定target接收的位置参数，用元组表示，默认不接收位置参数
    5. 参数 kwargs：用于指定target接收的关键字参数，用字典表示，默认不接收关键字参数
    6. 参数 daemon：用于指定线程实例对象是否是守护线程，默认不是守护线程

3. 子线程的创建2

  • 自定义继承自 Thread 的类对象：重写 __init__() 和 run() 方法，使用自定义类对象创建并启动子线程
  • 应用：根据自定义的类对象创建线程实例对象，然后调用线程实例对象的方法 start() 启动线程（调用方法 start() 后，会自动调用重写后的 run() 方法）
  • 与子线程的创建 1 的区别：
    • 在创建线程实例对象时无需再指定参数 target，参数 target 指定的函数的函数体转移到了方法 run() 中
    • 第 1 种方式创建线程实例对象时指定的其它参数，在第 2种方式中可以传递给重写后的特殊方法__init__()

二、进程

1. 自动创建与启动的进程

  • 简介：当PyCharm中运行一个py文件时，PyCharm对应的进程会自动创建并启动一个新进程，该进程被称为父进程，其默认名称为Python。当文件运行结束时，进程也会随之结束

2. 子进程的创建1

为什么要在 if __name__ == ‘__main__‘: 条件下执行代码？

解释：当在 Python 中使用多进程编程时，由于操作系统在创建子进程时会复制主进程的内存空间，包括所有已导入的模块和全局变量，当子进程被创建时，它会重新执行整个脚本，导致导入的模块被重复导入，全局变量被重新初始化，这可能会导致不正确的执行结果。在 __name__ == ‘__main__‘ 的条件下执行代码的作用是仅当你的脚本直接运行时（即作为主程序执行时），才会执行条件块中的代码，如果你的脚本被作为模块导入到其他脚本中，那么条件块中的代码就不会执行。

  • 使用类对象 Process 创建并启动子进程：标准库模块 multiprocessing 中提供了一个类对象 Process，用于表示进程

  • 应用：根据类对象 Process 创建进程实例对象，调用进程实例对象的方法 start() 即可启动进程（调用方法 start() 后，先会自动调用 Process 类中的 run() 方法，方法 run() 又会自动调用参数 target 指定的函数）

  • 创建进程实例对象时必须指定关键字实参：

    • 其构造方法定义为：__init__(self, group=None, target=None, name=None, args=(), kwargs={}, *, daemon=None)
    1. 参数 group：用于指定进程实例对象所属的进程组，默认不属于任何进程组
    2. 参数 target：用于指定被方法run()调用的函数，默认没有函数被调用
    3. 参数 name：用于指定创建的进程实例对象的名称，第n个子进程的默认名称为’Process-n’
    4. 参数 args：用于指定target接收的位置参数，用元组表示，默认不接收位置参数
    5. 参数 kwargs：用于指定target接收的关键字参数，用字典表示，默认不接收关键字参数
    6. 参数 daemon：用于指定进程实例对象是否是守护进程，默认不是守护进程

3. 子进程的创建2

  • 自定义继承自 Process 的类对象：重写 __init__() 和 run() 方法，使用自定义类对象创建并启动子进程

  • 应用：根据自定义的类对象创建进程实例对象，调用进程实例对象的方法 start() 启动进程（调用方法 start() 后，会自动调用重写后的 run() 方法）

  • 与子进程的创建 1 的区别：

    • 在创建进程实例对象时无需再指定参数 target，参数 target 指定的函数的函数体转移到了方法 run() 中

    • 第 1 种方式创建进程实例对象时指定的其它参数，在第 2 种方式中可以传递给重写后的特殊方法__init__()

4. 子进程的创建3

  • 标准库模块 os 中提供了一个函数 fork()：用于将当前进程复制一份为子进程，而后父进程和子进程从调用 fork() 处开始分叉，fork的含义即兵分两路，继续并行运行后面的程序
  • 与普通函数不同的是，函数 fork() 会返回两次，分别在父进程和子进程内返回。返回值有三种情况：
    1. 返回值小于0，表示复制子进程失败
    2. 返回值等于0，表示处在子进程中
    3. 返回值大于0，表示处在父进程中，返回值就是子进程的 id
  • 注意：函数 fork() 不是跨平台的，所以在 windows 操作系统上无法调用函数 fork()，而模块 multiprocessing 是跨平台的

三、进程池与线程池

1. 进程池Pool

  • 标准库模块 multiprocessing 中提供了一个类对象Pool：用于表示进程池，进程池中所能容纳的进程数目可以在创建 Pool 实例对象时进行指定，如果不指定，默认大小是 CPU 的核数
  • 作用：如果并发的任务数过多，一次性创建并启动大量的进程会给计算机带来很大的压力，那么就可以使用进程池对创建与启动的进程进行限制和管理（进程池中所能容纳的进程数目是固定的）
  • Pool 进程池的常用方法：
    • apply_async()：表示异步，子进程接收到请求之后就各自去执行了。作用与方法 start() 类似，但创建并启动的是由进程池管理的子进程
    • apply()：表示同步，子进程们将一个一个的执行，后一个子进程的执行永远以前一个子进程的结束为信号，开始执行
    • close()：关闭进程池，至此进程池中不再有进程可以接受任务（调用方法 join 之前必须先调用方法 close）
    • terminate()：结束当前进程池中的所有进程，不再处理未完成的任务，不管执没执行
    • join()：阻塞主进程，等待子进程执行完毕，再继续执行主进程
  • 各方法执行顺序：close() > terminate() > join()。其中，terminate和join方法可以都不执行，那么子进程和主进程就各自执行各自的，无论执行到哪里，子进程会随着主进程的结束而结束

2. ProcessPoolExecutor

  • 标准库模块 concurrent.futures 中提供了一个类对象 ProcessPoolExecutor：也用于表示进程池，与 Pool 相比，ProcessPoolExecutor 的功能和性能更加强大
  • 应用：类对象 ProcessPoolExecutor 遵守了上下文管理协议，使用 with 语句在离开运行时上下文会自动调用 ppe.shutdown(wait=True) 方法关闭进程池
  • ProcessPoolExecutor 进程池的常用方法：
    • submit(func, *args, **kwargs)：用于将单个任务提交到进程池，并返回一个 Future 对象，该对象代表了将来会完成的任务，此后会创建并启动由进程池管理的子进程
    • map(func, iterable, timeout=None)：用法与内置函数 map 类似，是阻塞的，用于批量提交多个任务到进程池，并返回一个迭代器，通过迭代器可以获取每个任务的结果。其第二个参数为可迭代对象，此后会创建并启动由进程池管理的子进程
    • shutdown()：阻塞主进程，待进程池管理的所有子进程执行完毕后，父进程再从被阻塞的地方继续执行，并且把进程池关闭（相当于 close + join 方法的组合）
  • Future对象：在 ProcessPoolExecutor 进程池中，方法 submit() 的返回值是一个 Future 实例对象，表示子进程所调用的函数的执行，可以调用 Future 的方法 result() 得到这个函数的返回值
    • result()：方法 result() 是一个同步方法，也就是每直到这个函数执行完毕之后，result() 才会返回
    • cancel()：任务取消
    • done()：获取当前 future 对象是否完成任务的状态

3. 线程池ThreadPool

  • 第三方库 threadpool 中提供了一个类对象 ThreadPool：用于表示线程池，线程池中所能容纳的线程数目可以在创建 ThreadPool 实例对象时进行指定，如果不指定，默认大小是 CPU 的核数
  • 作用：如果并发的任务数过多，一次性创建并启动大量的线程会给计算机带来很大的压力，那么就可以使用线程池对创建与启动的线程进行限制和管理（线程池中所能容纳的线程数目是固定的）
  • ThreadPool 线程池的常用方法：
    • makeRequests()：创建需要线程池处理的任务，返回一个列表
    • putRequest()：将需要线程池处理的任务全部交给线程池，此后会创建并启动由线程池管理的子线程
    • wait()：阻塞主线程，待线程池管理的所有子线程执行完后，父线程再从被阻塞的地方继续执行
  • 注意：wait() 方法等价 join() 方法。虽然线程中也可以使用 join() 方法，但那是 threading 模块中的 Thread 类对象中才有的方法，而此处 threadpool 模块中的 ThreadPool 类对象中只有 wait() 方法用于阻塞主线程

4. ThreadPoolExecutor

  • 标准库模块 concurrent.futures 中还提供了一个类对象 ThreadPoolExecutor：用于实现线程池，管理并发执行的任务

  • 应用：类对象 ThreadPoolExecutor 遵守了上下文管理协议，使用 with 语句在离开运行时上下文会自动调用 tpe.shutdown() 方法关闭线程池

  • 其余用法：线程池 ThreadPoolExecutor 与进程池 ProcessPoolExecutor 的所有用法几乎一致

5. 方法wait与as_completed

  • **wait(fs, timeout=None, return_when=ALL_COMPLETED)**：
    • 作用：用于等待一组任务完成，是阻塞的，可接受一个或多个 Future 对象作为参数，一直等待直到所有指定的 Future 对象都满足指定的条件
    • 参数 fs：是一个 Future 对象的列表，表示要等待的任务集合
    • 参数 timeout：指定最长等待时间，在超时时间内等待，超时则不再等待（如果指定为 None 或不指定，则一直等待）
    • 参数 return_when：指定何时返回，取值为FIRST_COMPLETED（当第一个Future实例对象已经完成或已被取消时）、FIRST_EXCEPTION（当第一个Future实例对象抛出异常时）和ALL_COMPLETED（当所有Future实例对象已经完成或已被取消时）
    • 返回值：返回两个集合组成的一个元组，第一个集合包含已经完成或已被取消的所有 Future 实例对象，第二个集合包含没有完成并且没有被取消的 Future 实例对象
  • **as_completed(fs, timeout=None)**：
    • 作用：用于获取一组任务中已完成的任务的结果，是非阻塞的，类似于生成器，它接受一个 Future 对象的序列作为参数，并返回一个迭代器，通过遍历迭代器可以立即获取已完成的任务的结果
    • 参数fs：是一个 Future 对象的序列，表示要获取结果的任务集合
    • 参数timeout：指定最长等待时间，在超时时间内获取已完成的任务，超时则不再等待（如果指定为 None 或不指定，则不会超时）
    • 返回值：返回一个 Future 实例对象的迭代器，可以通过 for 循环遍历获取已完成的任务的结果

四、多进程与多线程

1. 执行的不确定性

  • 简介：无论是多进程还是多线程，在默认情况下，多个进程或者多个线程的执行顺序和时间都是不确定的，完全取决于操作系统的调度

2. 守护属性daemon

作用：防止子进程/子线程因为任务无法退出造成整个程序卡死（作用于创建子进程或子线程的对象上）

解释：只要主程序的代码走到了最后一行，不管里面的子线程或子进程的任务有没有完成，都会随着主程序的退出而退出

  • 简介：daemon 属性用于标识线程或进程是否是守护线程或进程，其作用是控制线程或进程的生命周期（守护线程或进程是指在主程序结束时会被强制退出的子线程或子进程）
  • 作用：
    • 守护线程或进程的退出时机：当主程序（主线程或主进程）结束时，会等待所有非守护线程或进程完成任务后再退出。但是对于守护线程或进程，不管任务是否完成，都会被强制退出，不会阻塞主程序的退出
    • 资源清理：守护线程或进程在被强制退出时，不会执行清理操作，比如释放资源或关闭文件。因此，它们通常用于执行一些不需要在程序退出时做清理的任务

3. 阻塞方法join()

  • 简介：join() 方法用于等待一个多进程或多线程的任务完成，常用于让主线程或主进程等待指定的子线程或子进程执行完毕，然后再继续执行主线程或主进程的后续代码
  • 作用：等待任务完成和防止并发问题。在调用方法 join() 时，可以指定参数 timeout，从而指定子进程阻塞父进程的时间

4. 全局变量访问与共享

  • 全局变量在多个进程中不能共享：每个进程都有独立的内存空间，从而子进程的操作不会影响到父进程中的全局变量，它们不是共享同一块内存
  • 线程之间共享全局变量：子线程可以访问和修改父线程定义的全局变量，因为它们在同一进程内，共享同一块内存空间。默认情况下，所有线程都共享相同的全局变量
  • 注意：在子进程中修改全局变量，对父进程中的全局变量没有影响，因为子进程中只是对父进程中的全局变量做了一份拷贝，包括global都被拷贝了的，子进程与父进程中的全局变量是完全不同的两个变量

5. 操作数据的不安全性

  • 简介：多进程与多线程操作共享数据是不安全的，由于多进程或多线程的执行是不确定的，导致多进程或多线程操作共享数据的结果是不可预期的，常被称为不安全的

6. 进程间通信之共享内存

  • 简介：想要实现进程之间的通信，共享内存是常见的实现方式之一，它允许多个进程直接访问同一块内存（共享内存中对象的类型必须是ctypes的，ctypes是C语言兼容的数据类型）
  • 实现函数：
    • **Value(self, typecode_or_type, *args, lock=True)**，返回值表示一个数值
    • **Array(self, typecode_or_type, size_or_initializer, *, lock=True)**，返回值表示一个数组
    • 参数：typecode_or_type用于指定数值/数组中元素的类型码或ctypes类型，size_or_initializer用于指定数组的长度或Python中的序列

7. 进程间通信之队列

  • 简介：队列是一种线程安全的数据结构，可用于在不同的进程之间成传递数据和消息
  • 注意：进程池中数据传递是通过 Future 对象异步获取任务的结果，进程间的数据传递更为抽象，不直接使用队列
  • 方法：
    • put()：将数据放入队列中，是阻塞的。如果队列已满，put()操作会等待直到队列有足够的空间来放入数据
    • get()：从队列中获取数据，是阻塞的。如果队列为空，get()操作会等待直到队列中有数据可供获取
    • get_nowait()：从队列中获取数据。如果队列为空，则不会等待而是直接抛出异常
    • empty()：判断队列是否为空，返回布尔值
    • full()：判断队列是否已满，返回布尔值

五、并发同步

1. Lock锁

  • 简介：锁是一种同步机制，用于控制多个线程或进程对共享资源的访问。当多个线程或进程同时访问共享资源时，可能会导致数据不一致或错误的结果，为了确保在任何时候只有一个线程或进程可以访问共享资源，可以使用锁（Lock）来同步线程或进程的访问
  • 锁的两种状态：锁定（被某个线程或进程占用）和未锁定（可供其他线程或进程使用）
  • 注意：过度使用锁可能会导致性能下降，因为包含锁的相关代码只能以单线程模式执行，因此效率会大大降低
  • 应用：标准库模块multiprocessing和threading中均提供了一个类对象Lock，用于表示锁，以实现多进程或多线程之间的同步。由于类对象Lock遵守了上下文管理协议，所以可以使用with语句进行简化，在进入运行时上下文时会自动调用方法acquire()，在离开运行时上下文时会自动调用方法release()

2. 死锁

  • 简介：如果有多个共享数据，在多个线程或进程操作这多个共享数据时，如果两个线程或进程分别通过加锁占有一部分共享数据，并且同时等待对方释放锁，这样就会导致两个线程或进程永远相互等待而产生死锁，即当多个线程或进程同时持有一些资源并且试图获取其他资源时，可能会发生死锁
  • 死锁产生的必要条件：
    1. 互斥条件：至少有一个资源同时只能被一个线程或进程占用
    2. 请求与保持条件：线程或进程已经持有了一个资源，同时又请求其他资源
    3. 不可剥夺条件：资源只能在持有者释放时才能被其他线程或进程获取，不能被强制性地剥夺
    4. 环路等待条件：存在一组线程或进程，每个线程都在等待下一个线程所持有的资源
  • 避免死锁：
    1. 加锁顺序：强制线程按照特定的顺序获取锁，从而消除环路等待条件
    2. 超时：为获取锁设置超时时间，一段时间后如果未获取到锁就放弃或进行其他处理
    3. 资源分级：将资源分为不同的级别，要求线程只能按照级别顺序获取资源，避免相互等待
    4. 剥夺资源：在适当的时机，强制剥夺一些线程的资源，使得其他线程可以继续执行

3. RLock锁

  • 引出：在同一线程中，当调用了普通锁Lock的方法acquire()之后，如果在调用方法release()之前再次调用了方法acquire()，则会导致死锁
  • 简介：可重入锁 RLock 是一种同步机制，它允许同一个线程在持有锁的情况下多次获取同一个锁，以避免死锁，而其他线程必须等待该线程释放所有对锁的持有，主要用于解决递归函数调用中的同步问题。RLock继承自普通锁Lock，并在其基础上增加了重入的特性
  • 应用：标准库模块threading和multiprocessing中提供了一个用于表示锁的类对象RLock（Reentrant Lock，可重入锁）。同时类对象RLock也遵循了上下文管理协议

六、协程

协程：协程是一种特殊的函数，在同一线程内，它可以暂停执行以允许其他代码运行，然后再从暂停的地方继续执行（协程运行状态于多线程类似）

协程思维：一处异步，处处异步，语法版本需统一，3.6语法与3.7语法不可以混用

协程优点：

  • 无需线程上下文切换，协程避免了无意义的调度，可以提高性能
  • 无需原子操作锁定及同步开销
  • 方便切换控制流，简化编程模型
  • 高并发 + 高扩展性 + 低成本，一个CPU甚至可以支持上万的协程，适合用于高并发处理

协程缺点：

  • 进行阻塞操作（如IO时）会阻塞掉整个程序
  • 无法利用多核资源，协程的本质时单线程，不能同时将单个CPU的多个核用上，协程需要进程配合才能运行在多CPU上

最原始的协程：

1. asyncio模块

  • 简介：asyncio是Python标准库中的一个模块，用于编写异步、非阻塞的并发代码。asyncio的编程模型是一个消息循环，我们从asyncio模块中直接获取一个事件循环EventLoop的引用，然后把需要执行的协程扔到EventLoop中执行，以实现异步IO
  • 事件循环EventLoop：事件循环是asyncio的核心，负责处理和调度异步任务和事件。程序会开启一个死循环，将一些函数注册到事件循环上，当满足事件发生的时候，调用相应的协程函数。事件循环的切换底层依靠io耗时判断，当判断出某个任务是比较耗时的，则暂停当前任务切换到另一个任务
  • 协程Coroutines：协程是一种特殊的函数，协程可以在执行过程中暂停，并在需要时恢复执行。协程函数的调用不会立即执行函数，而是返回一个协程对象，协程对象需要注册到事件循环，由事件循环调用
  • Task任务对象：任务是协程的包装器，用于在事件循环中并发运行协程。任务对象允许跟踪协程的状态，等待它完成或取消它
  • Future对象：用于表示尚未完成的异步操作的结果，允许等待和处理异步操作的完成状态，以及控制异步操作的结果或异常，通常与await一起使用。Task任务对象是Future的子类，任务对象Task具有更多的功能，可以代表协程的执行状态和结果，因此它们是一种特殊的Future
  • async/await关键字：async关键字用于定义协程，声明这是一个协程函数。await关键字用于挂起协程的执行，等待异步操作完成，通常用于调用其他异步函数或操作

协程函数：协程函数是使用async def关键字定义的特殊函数，它可以在执行过程中暂停和恢复执行，通常用于定义异步任务的逻辑

协程对象：协程对象是通过调用协程函数来创建的，它实际上是协程函数的实例，表示一个可执行的异步任务，可以添加到事件循环中并执行

2. asyncio.run()

  • 作用：用于运行一个协程，是Python 3.7引入的新方法
  • 用法：接受一个协程对象（不能是任务对象）作为参数，此函数总是创建一个新的事件循环并运行协程，然后关闭事件循环
  • 返回值：即为协程函数的返回值，如果协程函数没有返回值，则返回None

3. asyncio.sleep()

  • 作用：挂起当前协程的执行，模拟等待一段时间
  • 用法：接受一个浮点数或整数作为参数，表示等待的秒数
  • 返回值：返回一个Future对象，表示异步操作的结果

4. await关键字

  • await关键字：await 是用于等待异步操作完成的关键字，其作用是在遇到耗时任务时让出控制权，让事件循环去执行其他任务，从而保持整个程序的异步执行
  • 特点：
    1. 只能在异步函数内使用：await 只能在异步函数内使用，这些函数使用 async def 声明，函数内部的耗时任务必须通过 await 修饰（await 可以在一个异步函数内出现多次）
    2. 等待异步操作完成： await 后面通常跟着一个可等待对象，如协程对象、Task 对象或 Future 对象
  • await可以将任务进行提交到事件循环中，只能提交可等待对象 [coroutine, task, future]：
    • 将多个任务分次提交到事件循环中：此时事件循环始中终保持着只有一个任务的状态，每一个任务的执行都要等待上一个 await 等待完毕并得到返回值，才能将下一个任务添加到事件循环，所以是同步执行
    • 将多个任务一次性全部提交到事件循环中：通常使用 asyncio.wait() 或 asyncio.gather() 等待一组任务完成，使用 await 将任务一次性全部提交到事件循环中，以并发执行
  • 注意：Task 对象和 Future 对象本身就是可以被等待的可等待对象

5. task对象

  • 不同版本的方法：

    • asyncio.create_task()：会自动获取当前存在的事件循环（python 3.7之后的版本使用）
    • loop.create_task()：需要使用当前的loop事件循环对象驱动（python 3.6之前的版本使用）

  • 作用：用于创建一个任务，将协程包装为一个任务对象，以便在事件循环中并发运行（支持并发执行！）

  • 用法：接受一个协程对象作为参数，将协程包装，返回一个任务对象

6. asyncio.wait()

  • 作用：等待一组Future对象完成，并且可以获取被提交的任务的运行状态和返回值（异步执行任务，同步获取结果）
  • 用法：参数接受一个可迭代的Future对象集合（不能接受协程对象或者协程对象集合）
  • 返回值：返回一个包含两个元素的元组
    • done：包含已经完成的任务对象的集合
    • pending：包含未完成和正在执行的任务对象的集合

7. asyncio.gather()

  • 作用：同时提交多个任务（tasks或coroutines）到事件循环，并等待它们全部完成后获取结果，这种等待是同步的（异步执行任务，同步获取结果）
  • 用法：接受单个或多个协程对象或future对象作为参数，多个对象可以直接传入，但不能作为迭代对象传入
  • 返回值：返回一个包含了所有协程结果的列表，结果的顺序与传入的协程顺序一致

8. asyncio.as_completed()

  • 作用：返回一个异步生成器（async generator），按照完成的顺序产生Future对象，一旦某个任务完成，就会立即返回它，而不需要等待其他任务完成
  • 用法：使用async for异步循环来迭代这个生成器，获取已完成的Future对象
  • 返回值：返回一个按完成顺序产生Future的生成器，通过迭代生成器，一旦某个任务完成，就会立即返回一个可await的任务对象，然后可以使用await获取任务的结果

七、协程进阶

1. asyncio的future对象

  • 简介：future类是task类的基类，task对象只有运算得到返回值后，await的对象才能传回值并且向下运行，这个功能是future对象来实现并提供的。future源码底层中存在一个_state状态，一旦_state状态值变成finished，await就不再继续等待（解堵塞）
  • 用法：由于future对象偏向于底层（常用于python 3.5版本），新版本中一般不直接使用，而是使用task对象

2. asyncio.ensure_future()

  • 作用：类似asyncio.create_task()也是将一个协程包装为一个任务对象（python 3.6之前的版本使用，兼容高版本）
  • 用法：接受一个协程对象作为参数，返回一个任务对象

3. 线程池中的future对象

  • 回顾线程池与进程池中的future对象，具体参考本篇中的三、进程池与线程池

4. 线程与协程交叉编程

  • 引入：在编程时，遇到某个第三方模块不支持asyncio协程异步的时候，则需要线程、进程和协程的两种Future交叉使用。而一般情况下，不会交叉使用，要么使用asyncio协程式实现异步，要么统一用进程或线程池来实现异步
  • 案例：crm项目中80%是基于协程异步编程 + MySQL，这种情况下只有MySQL内部也支持async异步，两者才能实现无缝衔接。假如MySQL不支持协程，则需要考虑对此用线程、进程来实现了
  • loop.run_in_executor()实现原理：
    1. 在线程池中，先调用线程池的submit()方法去线程中申请一个func函数，并返回一个Future对象（这里的Future对象与asyncio协程没有任何关系，是concurrent.futures.Future）
    2. 调用futures.wrap_future()方法将线程中的Future对象（concurrent.futures.Future）包装成协程中的Future对象（asyncio.Future）

5. 自定义异步迭代器类

  • 基本结构：
    • **__aiter__()**： 必须返回一个异步迭代器对象自身。它不需要使用 async 关键字，因为它通常不执行异步操作，只是返回对象本身
    • **__anext__()**：负责异步获取下一个元素，并在迭代结束时引发 StopAsyncIteration 异常。它通常包含异步操作
  • 用法：使用自定义的异步迭代器，可以在异步函数中创建异步迭代器对象，并使用异步for循环来遍历元素

6. add_done_callback

  • 简介：add_done_callback是Python中 asyncio.Future 对象的一个方法，主要作用是在异步操作完成后执行一个回调函数
  • 作用：用于添加一个回调函数（callback），当该Future对象完成时，会自动调用这个回调函数
  • 特点：
    1. 允许在异步操作完成后执行自定义逻辑，这对于处理异步操作的结果或执行清理操作非常有用
    2. 其添加的回调函数是非阻塞的，因此不会影响主事件循环的执行
  • 注意：定义回调函数时需定义为普通函数，而不是协程函数，不能用async修饰该函数

7. 异步上下文管理器

  • 基本结构：
    • **__aenter__**：在进入异步上下文时执行，通常用于获取异步资源，例如打开文件或建立网络连接。它可以包含异步操作，因此需要async修饰函数为协程函数，并且函数内需要使用到 await 关键字
    • **__aexit__**：在退出异步上下文时执行，通常用于释放异步资源，例如关闭文件或断开网络连接。它也可以包含异步操作，因此需要async修饰函数为协程函数，并且函数内需要使用到 await 关键字
  • 用法：使用 async with 语句创建异步上下文和使用异步上下文管理器，在其中执行需要异步资源的操作

8. uvloop

  • 简介：在 Python 中进行异步编程时，事件循环的性能和效率对于整体应用程序的性能至关重要。uvloop是一个第三方库，它提供了一个高性能的事件循环实现，专门为 Python 异步编程而设计

  • 特点：

    • 高性能事件循环：uvloop基于libuv，这是一个高性能的事件驱动库，它可以提供非常快速的异步I/O操作。相对于标准的asyncio事件循环，uvloop提供了更好的性能，特别是在高负载和高并发情况下
    • 100%兼容asyncio：uvloop是asyncio的替代事件循环实现，它与标准库中的asyncio完全兼容。这意味着你可以将现有的asyncio代码迁移到uvloop，并且不需要修改你的代码，只需将事件循环的实现切换为uvloop即可
    • 协程和异步代码性能提升：由于uvloop提供了更快的事件循环，因此它可以改善协程和异步代码的性能，特别是在处理大量并发操作时

  • 用法：使用 asyncio.set_event_loop_policy(uvloop.EventLoopPolicy()) 可以将uvloop设置为默认的事件循环策略，是全局的，所以不能放在main函数中
