2. MuMu 模拟器抓包配置

  • mac 完整环境配置（无可用证书）：

    1. 通过 App Cleaner 卸载 MuMu 模拟器 Pro，重新安装破解版，运行前执行 sudo spctl --master-disable + sudo xattr -d com.apple.quarantine + sudo xattr -d com.apple.quarantine /Applications/MuMuPlayer.app
    2. 打开 MuMu 模拟器 Pro，创建安卓手机，设置权限 可写系统盘 + 开启手机 Root 权限，并启动
    3. 从抓包软件（Proxyman、Charles）中下载 .pem 证书，在模拟器中打开开发者 –> 打开 ADB (16416, 5555)
    4. 执行命令 exit + cd /Applications/MuMuPlayer.app/Contents/MacOS/MuMuEmulator.app/Contents/MacOS/tools/ + openssl x509 -subject_hash_old -in /Users/xfblog/Desktop/代理.pem证书 输出一串 87d7a3db 混合中英文字符串
    5. 将 .pem 证书重新命名为此字符串 mv /Users/xfblog/Desktop/代理.pem /Users/xfblog/Desktop/87d7a3db.0
    6. 用 ADB 把证书放到系统目录，执行 ./adb -s emulator-5554 push /Users/xfblog/Desktop/87d7a3db.0 /system/etc/security/cacerts + ./adb -s emulator-5554 shell "chmod 664 /system/etc/security/cacerts/87d7a3db.0"（如果报错有多个设备，则可通过 adb devices 查看所有设备，通过 -s emulator-5554 指定设备）
    7. 最后在模拟器中打开设置 –> 网络WIFI –> 右上角铅笔编辑 –> 代理选择手动代理 –> 填入电脑 IP 和端口（抓包工具代理端口）

  • 有可用证书：

    1. 打开 MuMu 模拟器 Pro，创建安卓手机，设置权限 可写系统盘 + 开启手机 Root 权限，并启动
    2. 从抓包软件（Proxyman、Charles）中下载 .pem 证书，在模拟器中打开工具 –> 打开 ADB (16416, 5555)
    3. 执行命令 exit + cd /Applications/MuMuPlayer.app/Contents/MacOS/MuMuEmulator.app/Contents/MacOS/tools/ + ./adb push /Users/xfblog/Desktop/87d7a3db.0 /system/etc/security/cacerts + ./adb shell "chmod 664 /system/etc/security/cacerts/87d7a3db.0"（异常由 adb devices 查看所有设备，通过 -s emulator-5554 指定设备）

  • 关于软件：直接在自己手机上下载，下载后分享至微信即为 apk 安装包

3. git 拉项目配置

  • 在电脑任意位置：生成 SSH 密钥：sudo ssh-keygen -t rsa -b 4096 -C "chuanfu7w@163.com"

    • 私钥文件：/var/root/.ssh/id_rsa
    • 公钥文件：/var/root/.ssh/id_rsa.pub
    • 密钥指纹：SHA256:R9/3+LprMgYMrIBTFw3oeeHAo4m/G8fJB8MqvRD15rw
    • 随机艺术图：这个图形是用来可视化你的公钥特征的

  • 移动公钥和私钥到用户文件中，并配置权限：

    1. sudo mv /var/root/.ssh/id_rsa /Users/xfblog/.ssh/
    2. sudo mv /var/root/.ssh/id_rsa.pub /Users/xfblog/.ssh/
    3. chmod 600 /Users/xfblog/.ssh/id_rsa
    4. chmod 644 /Users/xfblog/.ssh/id_rsa.pub
    5. cat /Users/xfblog/.ssh/id_rsa.pub

  • 在 github 中添加 SSH 密钥：

    1. 点击右上角的头像，选择 Settings（设置）
    2. 在左侧菜单中，点击 SSH and GPG keys
    3. 点击 New SSH key 按钮
    4. 在 Title 中输入一个描述（如“我的本地机器”）
    5. 在 Key 中粘贴你刚才复制的公钥内容
    6. 点击 Add SSH key 保存

  • 配置权限：

    1. 查看文件的拥有者：ls -l /Users/xfblog/.ssh/id_rsa
    2. 更改文件的拥有者：sudo chown xfblog /Users/xfblog/.ssh/id_rsa
    3. 确保 SSH 代理运行并添加密钥：eval "$(ssh-agent -s)"，ssh-add /Users/xfblog/.ssh/id_rsa
    4. 测试 SSH 连接：ssh -T git@github.com

  • git remote set-url origin git@github.com:Chuanfu-Wei/xfblogProjects.git

  • 安装步骤：brew install pyenv
  • 环境配置：

  • 使用方法：

    1. 安装 python 版本：pyenv install 3.9.18

    2. 列出所有可安装的 Python 版本：pyenv install --list

    3. 查看已安装的 Python 版本：pyenv versions

    4. 设置全局版本：pyenv global 3.9.18

    5. 设置本地版本：pyenv local 3.9.18

    6. 验证 Python 版本：python --version

  • 虚拟环境配置：

    • 安装 pyenv-virtualenv：brew install pyenv-virtualenv

      在 .zshrc 或 .bashrc 中添加配置：

      重新加载配置文件：source ~/.zshrc

      创建虚拟环境：pyenv virtualenv <python版本> <虚拟环境名称>

      列出所有虚拟环境：pyenv virtualenvs

      激活虚拟环境：pyenv activate <虚拟环境名称>

      停用虚拟环境：pyenv deactivate

      设置全局虚拟环境：pyenv global <虚拟环境名称>

      设置本地虚拟环境：pyenv local <虚拟环境名称>

      删除虚拟环境：pyenv uninstall <虚拟环境名称>

      查看所有 Python 版本和虚拟环境：pyenv versions
