算法总结图

一、哈希摘要算法

哈希摘要：是一种通过将数据应用哈希函数来生成固定长度的数据摘要的过程

哈希摘要算法的基本特点包括：

  1. 不可逆性：数据不能被逆向还原，即无法从散列值（哈希摘要）反推出原始数据
  2. 固定长度输出：不同输入长度的数据都生成相同长度的摘要
  3. 唯一性：不同的输入数据应生成唯一的摘要，但碰撞是理论上可能的，即两个不同的输入生成相同的摘要
  4. 快速计算：对于给定输入，摘要的生成应该是高效的

1. MD5

MD5（Message-Digest Algorithm）：是一种广泛使用的哈希算法（密码散列函数），可以产生出一个128位（16字节）的散列值，用于确保信息传输完整一致。MD5由美国密码学家罗纳德·李维斯特（Ronald Linn Rivest）设计，于1992年公开，用以取代MD4算法。这套算法的程序在 RFC 1321 标准中被加以规范。1996年后该算法被证实存在弱点，可以被加以破解，对于需要高度安全性的数据，一般建议改用其他算法，如SHA-2。2004年，证实MD5算法无法防止碰撞（collision），因此不适用于安全性认证，如SSL公开密钥认证或是数字签名等用途。

MD5特征：32长度、十六进制

md5应用领域：

  1. 密码保护：前端后端密码对比，不怕数据库数据丢失
  2. 文件完整性校验：确保文件传递的前后一致
  3. 数字签名：防止发布的软件被篡改
  4. 云盘秒传：云盘有时并不会把全部的数据上传，而是拿到数据加密之后的md5去文件数据库对比

2. SHA

SHA（Secure Hash Algorithm）：是由美国国家标准与技术研究院（NIST）制定的一系列哈希算法标准，包括 SHA-1、SHA-256、SHA-384、SHA-512 等。旨在产生一种固定长度的哈希值，通常以十六进制表示。SHA 算法家族被广泛用于数字签名、数据完整性验证等安全应用。

SHA家族的特征（版本越强，密文越长，代价是速度越慢）：

  1. SHA-1：20字节，40长度
  2. SHA-224：28字节，56长度
  3. SHA-256：32字节，64长度
  4. SHA-384：48字节，96长度
  5. SHA-512：64字节，128长度

3. HMAC

HMAC（Hash-based Message Authentication Code）：是一种基于哈希函数的消息认证码算法，结合了哈希函数和密钥以提供消息完整性和真实性的验证。于1996年提出，并于1997年作为 RFC2104 被公布。

HMAC的基本思想：通过将消息和密钥混合在一起，并将其输入到哈希函数中，生成一个固定长度的摘要。这个摘要（HMAC）随着消息一起传递，并在接收端使用相同的密钥和哈希函数重新计算，但同样是不可逆的，以验证消息的完整性（可以与各种哈希算法捆绑使用，并拥有相同的特点）

二、对称加密算法

对称加密算法的基本思想：使用相同的密钥（也被称为秘密密钥）同时执行加密和解密操作，也就是说加密和解密用的是同一个算法。这意味着在通信的两端，发送方和接收方都必须共享相同的密钥。对称加密算法通过这个共享的密钥来隐藏和还原原始数据，确保只有持有密钥的人能够解密和访问数据。

常见的对称加密算法：DES、AES（Advanced Encryption Standard*）、3DES、RC4、IDEA

对称加密的特点：可逆性、密钥共享、高效性、适用于随机访问

1. 算法工作模式

  • ECB模式（电子密码本模式）：
    • 基本思想：将明文切分成固定大小的块，然后对每个块独立进行加密。每个块使用相同的密钥进行加密，且每个块的加密是独立的，不依赖于其他块，最后将每一组加密的结果进行拼接得到密文（最简单）

  • CBC模式（密码分组链接模式）：

    • 基本思想：引入了初始化向量 IV，在第一个块中先让 IV 与明文进行异或操作，再进行加密生成密文，此时将密文又作为下一个块的 IV 与下一个块的明文加密生成下一个块的密文，以此类推（使用最多）
  • CFB模式（密码反馈模式）：
    • 基本思想：吸收了流加密的特点，可以理解为在CBC模式中实现了流加密。加密关键是使用前一次加密的输出作为下一次加密的输入。将前一个密文块输入（反馈）到加密函数中，产生一个伪随机流，将伪随机流与当前块的明文异或，得到密文，再将密文进行位移处理后作为下一块的输入，以此类推
  • OFB模式（输出反馈模式）：
    • 基本思想：类似于CFB，但反馈的是加密函数的输出，使用上一个块的加密函数的输出与当前块的明文进行异或操作，得到密文
  • CTR模式（计数器模式）：
    • 基本思想：与CFB、OFB模式为同一类，但加密关键是使用计数器和密钥生成的伪随机数流与明文进行异或操作，得到密文。计数器的值逐块递增，每个块都使用不同的计数器值进行加密，适用于流加密，支持并行处理（可以代替CFB、OFB模式）

2. DES

DES（Data Encryption Standard）：是一种对称加密算法，最早由IBM于1977年提出并被美国国家标准局（NIST）采纳为联邦信息处理标准（FIPS）。由于DES密钥较短，在1999年被AES（高级加密标准）取代。

加密原理：DES属于对称密码算法中的分组加密算法（块加密），原理是将明文分成n个64位的字节块，分别进行加密，当明文块不足64位时会通过双方统一的填充方案进行填充（PKCS5Padding）。密钥长度64位，56位参与运算，8位校验位。每个明文数据块都经过 des 加密后，所有密文块串联在一起形成完整的密文。

  • DES加密算法的主要步骤：

    1. 密钥生成：
      • DES使用56位的密钥，但实际上是使用64位的密钥，其中有8位是奇偶校验位。这样实际有效的密钥长度为56位
      • 密钥被分为16个子密钥，每个子密钥包含48位
    2. 初始置换（IP置换）：
      • 明文（64位）被初始置换，即重新排列其位，产生一个初始置换的数据块
    3. 分为左右两半：
      • 初始置换后的数据块分为左右两个32位的半块
    4. DES采用Feistel网络结构。每一轮的右半块通过一个轮函数进行处理，这个轮函数包括：
      • 扩展（Expansion）： 将右半块的32位扩展为48位
      • 密钥混合（Key Mixing）： 将扩展后的右半块与子密钥进行异或运算
      • S盒代替（S-box Substitution）： 将48位的数据分成8组，每组6位，通过8个S盒进行代替，每个S盒将6位输入映射到4位输出
      • P盒置换（Permutation）： 对S盒输出进行置换，得到最终的32位输出
    5. 左右交换：
      • 经过轮函数处理后，左半块与右半块进行异或运算，然后交换位置
    6. 重复16轮：
      • 上述的轮函数过程重复16轮
    7. 逆初始置换（Final Permutation）：
      • 最后一轮处理完后，左右两半进行逆初始置换，得到最终的密文

3. AES

AES（Advanced Encryption Standard）：全称高级加密标准，在密码学中又称 Rijndael 加密法，由美国国家标准与技术研究院 （NIST）于 2001 年发布，并在 2002 年成为有效的标准，是美国联邦政府采用的一种区块加密标准。这个标准用来替代原先的 DES，支持多种密钥长度，包括128位、192位和256位，目前已成为对称密钥加密中最安全、最流行的算法之一。

加密原理：AES属于对称密码算法中的分组加密算法（块加密），原理是将明文分成n个128位的字节块，分别进行加密，当明文块不足128位时会通过双方统一的填充方案进行填充（PKCS5Padding）。密钥的长度可以是128位、192位和256位。每个明文数据块都经过 aes 加密后，所有密文块串联在一起形成完整的密文。

  • AES加密算法的主要步骤：

    1. 轮密钥生成（Key Expansion）：
      • 根据输入的密钥长度（128位、192位或256位），生成扩展密钥。这些扩展密钥将在后续的轮函数中使用。
    2. 初始轮（Initial Round）：
      • 将明文与扩展密钥进行初始轮的加法运算。
    3. AES的核心是多轮处理（Rounds），每一轮包括以下四个步骤：
      • 字节替代（SubBytes）： 使用一个固定的8x8的替代盒（S盒），将每个字节替换为另一个字节。
      • 行移位（ShiftRows）： 对状态矩阵的每一行进行循环左移。第一行不移动，第二行左移一个字节，第三行左移两个字节，第四行左移三个字节。
      • 列混淆（MixColumns）： 利用矩阵运算对状态矩阵的每一列进行混淆。
      • 轮密钥加（AddRoundKey）： 将当前轮的轮密钥与状态矩阵进行按位异或运算。
    4. 最终轮（Final Round）：
      • 在最后一轮中，省略MixColumns步骤，只执行SubBytes、ShiftRows和AddRoundKey。
    5. 逆操作（Inverse Operations）：
      • 为了实现解密，需要对加密过程中的每个步骤进行逆操作。逆操作包括逆字节替代、逆行移位、逆列混淆和逆轮密钥加。

  • 注意：AES在不同的密钥长度下有不同的轮数

    • 10轮（128位密钥）
    • 12轮（192位密钥）
    • 14轮（256位密钥）

三、魔改算法

魔改（魔改算法）：通常是指对已有的算法进行修改或改进的过程。这个过程可能包括调整参数、改变某些步骤、添加新的元素，或者以其他方式对算法进行修改，以满足特定的需求或优化特定的性能指标。

  • 判断是否是魔改算法：观察算法库对象中的有没有一系列常用的标准算法，大致可以推断出是否是魔改算法，是则需要将与魔改算法库对象定义相关的所有代码都要扣下来，包括自执行方法（测试网站）

四、非对称加密算法

非对称加密算法的基本思想：对称加密使用相同的密钥进行加密和解密，而非对称加密使用一对密钥，包括公钥和私钥，分别用于加密和解密。如果用公钥加密信息，只有用私钥才能解密。反之亦然，用私钥加密的信息只能用对应的公钥进行解密。这种关系使得公钥可以安全地分享给其他人，而私钥则必须保持秘密。

常见的非对称加密算法：RSA（Rivest-Shamir-Adleman*）、DSA、ECC（椭圆曲线密码学）

可用于数字签名领域：验证消息的完整性和来源。发送方使用私钥对消息进行签名，而接收方可以使用发送方的公钥来验证签名。这确保了消息在传输过程中没有被篡改，并且确定是由私钥持有者生成的。

1. RSA

RSA（Rivest-Shamir-Adleman）：是一种非对称加密算法，于1977年提出。RSA算法基于数论的难题，特别是大数分解的难度，其安全性依赖于质数因子分解的困难性。在RSA中，相同的明文每次生成的密文值都是不一样的，每次加密都会采用不同的填充和随机数，以确保生成的密文在统计上看起来是随机的。

  • RSA加密算法的主要步骤：

    1. 密钥生成：

      • 选择两个大质数p和q：这两个质数越大，破解RSA就越困难。
      • 计算n：计算n = p * q，n是两个质数的乘积。
      • 计算欧拉函数φ(n)：计算φ(n) = (p-1)(q-1)。
      • 选择公钥e：选择一个满足1 < e < φ(n)且与φ(n)互质的数，通常选择65537。
      • 计算私钥d：计算满足(e * d) mod φ(n) = 1的私钥d。

      公钥是(n, e)，私钥是(n, d)。公钥可以公开，私钥必须保密。

    2. 加密过程：

      • Alice拥有Bob的公钥(n, e)。
      • Alice将明文M转换为整数m，确保0 < m < n。
      • Alice计算密文C，其中C ≡ m^e (mod n)。
      • Alice将密文C发送给Bob。

      加密过程利用了模幂运算，即将明文的整数形式m，通过取e次方并对n取模得到密文C。这个操作使得只有私钥持有者才能轻松解密消息。

    3. 解密过程：

      • Bob使用私钥(n, d)。
      • Bob计算明文m，其中m ≡ C^d (mod n)。
      • Bob将整数m转换为明文M。

      解密过程利用了模幂运算，与加密过程相反，通过将密文的整数形式C，取d次方并对n取模得到原始的明文。

    4. 安全性基础：

      • RSA的安全性建立在大整数分解的困难性基础上，即将n分解为其两个大质因数p和q。
      • 当n足够大时，对n进行分解变得非常耗时，因此使得破解RSA成为一项艰巨的任务。

    5. 性能注意事项：

      • 由于RSA的加密和解密操作相对耗时，通常不直接用于加密大量数据。通常，RSA用于安全地交换对称密钥，而对称密钥用于实际的数据加密。

五、SM国密系列算法

SM国密：SM国密算法系列是中国国家密码管理局（State Cryptography Administration，简称SCA）发布的一系列密码算法标准，旨在保障信息安全和国家安全。这一系列算法包括对称加密算法、非对称加密算法、数字签名算法等多个方面，其中SM2、SM3和SM4是其中最为重要的代表。

1. SM2

SM2（椭圆曲线公钥密码算法）：SM2是一种非对称加密算法，基于椭圆曲线密码学，采用了椭圆曲线数字签名算法（ECDSA）的变种，适用于数字签名、密钥交换等场景。SM2提供的安全性相当于RSA算法，该算法采用椭圆曲线上的点运算，计算速度更快，且使用了更短的密钥长度，提高了加密和解密的效率。

算法流程：

  1. 密钥生成：生成公私钥对
  2. 签名：利用私钥对消息进行签名
  3. 验证：利用公钥验证签名的合法性
  4. 密钥协商：两方利用各自的私钥和对方的公钥协商出共享密钥

2. SM3

SM3（密码杂凑算法）：SM3是一种哈希算法，用于生成固定长度256位的消息摘要，适用于数字签名、消息认证码等场景。SM3提供的安全性相当于SHA-256算法，输出长度为32字节即64长度，该算法在性能上更有优势，同时具有抗碰撞特性，保证了足够的安全性。

算法流程：

  1. 填充：将消息填充到满足特定要求的长度
  2. 消息扩展：将填充后的消息划分成512比特的消息块，并进行消息扩展
  3. 压缩：通过迭代压缩函数，对消息块进行处理
  4. 输出：最终得到256比特的消息摘要

3. SM4

SM4（分组密码算法）：SM4是一种对称加密算法，采用32位分组长度和128位密钥长度，用于块加密和流加密，适用于数据加密、数据完整性验证等场景。SM4提供的安全性相当于AES算法，该算法的设计参考了现代分组密码的设计原则，采用了Feistel网络结构，结合替代-置换网络（SPN）结构，使其在安全性和性能之间取得了良好的平衡，同时实现了相对高的加密和解密速度。

算法流程：

  1. 密钥扩展：生成轮密钥
  2. 初始置换：对输入数据进行初步置换
  3. 轮函数：采用非线性的S盒替代、线性的P盒变换等操作
  4. 轮运算：通过32轮迭代，实现加密或解密操作
  5. 逆初始置换：对输出数据进行逆置换，得到最终结果
