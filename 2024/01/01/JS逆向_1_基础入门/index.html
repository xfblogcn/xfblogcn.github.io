一、JS调试技巧

在页面中右击 dom 元素标签 –> 存储为全局变量 –> dir(temp1) –> 即可快速查看标签的一些属性（js 对象属性）

1. 浏览器开发者工具

  • 设置：偏好设置 –> 开启 “源代码/来源” 下的 “在匿名和内容脚本中搜索”
  • 应用面板：可以查看和删除 Cookie 信息
  • 源代码面板：
    1. 网页：当前网站加载的源代码位置
    2. 替换：替换网站源代码，实现访问网站时不走服务器所返回的文件，而是直接走自己替换的本地文件
    3. 替换步骤：网络面板 –> 选择加载的源代码文件右击 –> 在”来源”面板中打开 –> 右击文件 –> 替换内容 –> 授权 –> 更改代码 –> 保存刷新网页
    4. 代码段：可以编写多段测试代码

2. 断点定位

普通断点调试按钮：

  1. 暂停/继续执行脚本
  2. 不进入函数内部
  3. 进入函数内部
  4. 跳出当前函数
  5. 单步调试，适用于异步
  6. 关闭/启动断点（关闭后XHR断点处还是会停住）

日志断点：用于输出每一次经过断点时的某些数据，只输出数据，不截停代码。使用时不需要输出语句，直接写变量或字符串，多个用逗号隔开

条件断点：用于只断住某个数据在某个条件下的代码

  • DOM事件断点定位加密：

    • 已有事件断点：元素面板 –> 选择元素 –> 右侧”事件监听器” –> 选择事件定位js文件 –> 断点并调试（测试网站）
    • 添加元素断点：元素面板 –> 选择元素 –> 右击元素代码 –> “发生中断的条件” –> 选择事件

  • XHR请求断点定位加密：

    • 网络面板 –> 确认数据接口url –> 源代码/来源面板 –> 右侧”XHR/提取断点” –> 添加url域名后参数前的部分 –> 断点并调试（测试网站）

3. 方法栈（调用堆栈）

  • 方法栈：当 JavaScript 执行函数时，会在方法栈中创建一个新的帧（frame），表示该函数的执行上下文。方法栈就是由一系列帧组成的，每个帧对应一个函数的执行上下文，帧的数量取决于当前代码中函数的调用深度，这些帧按照函数调用的顺序排列，即先进后出，构成了调用堆栈

  • 解释：当一个函数被调用时，一个新的帧被推入方法栈的顶部，该帧包含了函数的执行上下文信息，例如局部变量、参数、返回地址等。当函数执行完毕时，对应的帧从方法栈中弹出，控制流回到调用该函数的地方

  • 错误追踪：当 js 代码执行过程中发生错误时，错误信息中会包含一个堆栈跟踪（stack trace），它显示了错误发生时方法栈的状态，其包括了函数调用的顺序，以及错误发生的地方

4. debugger 反调

反调手段：

  1. 无限debugger：使用无限 debugger 阻止爬虫工具的正常操作，会导致浏览器调试器无法正常工作
  2. 防止格式化：代码必须压缩成一行，通过 RegExp + toString() 检测代码是否被格式化
  3. 死代码注入：在检测到代码被格式化后，造成卡死的情况就是通过死代码注入实现的
  4. Web Worker 技术：加载 authProxy.html 页面执行 js 脚本用于禁止断点调试 <html debug="false">

  • 无限debugger解决办法（跟栈分析，置空栈底方法为主）：

    1. 浏览器自带：右击 debugger 处 –> “一律不在此处暂停” / “添加条件断点”（false）

    2. 置空调用方法：跟栈分析，找到源头的 debugger 方法是如何生成的，置空传入或调用的 debugger 方法

    3. 替换文件：把 js 文件保存到本地修改，修改范围主要是将 debugger 相关的代码删除或者改写

    4. 注入代码（Hook，特点是无限创造VM文件，造成浏览器卡死）：

  • 防止格式化解决办法：

    • 原理：一般是通过 RegExp正则匹配 + toString() 检测某些函数，是否有换行等格式化操作，返回布尔值
    • 解决办法：替换文件，将正则的处理逻辑修改为 true 值

  • 死代码注入解决办法：

    • 原理：通过两个函数互相无限制调用，或者直接通过 while 写死循环，即 while (!![]) {}
    • 解决办法：在正则检测返回为 false 时，继续往下调试，可以定位死代码注入的逻辑

  • 注意：关键字搜索一般是最里层的具体加密位置，通常需要看上几帧的栈，才是调用的位置

二、补环境

1. 基础补环境

补环境重点之导出关键方法为全局方法：通过 window.aaa 直接将方法赋值全局对象，后续可直接调用

  • 常见环境检测：

    1. exports 对象：在 Node.js 中 exports 是一个用于模块导出的对象，而在浏览器中是undefined
    2. 全局对象：在浏览器中的全局对象是 window，而在 Node.js 中的全局对象是 global

  • 常见环境检测的方法：

    • 通过 IF 判断语句走错误环境逻辑：使代码在 node 环境下走与浏览器不一致的逻辑分支，从而返回误差数据（解决：补完整环境，或者分析网页走的逻辑，与网页保持一致）
    • 通过 try 异常捕获不存在的环境：使代码缺少环境也不报错，从而逆向结果错误，返回错误数据（解决：直接输出异常的内容，根据异常补完整环境，或者直接把异常捕获的代码删除，将错误暴露出来）

  • jsdom 补环境：jsdom 是一个 js 库，用于在 node.js 环境中模拟类似于浏览器环境中的 DOM API（文档对象模型）

    • **环境安装：npm install jsdom --save、npm install canvas --save**（可能安装错误，官网下载）

  • selenium 补环境：创建一个本地 html 文件，将 js 代码写入，同时将需要调用的方法通过 window 对象导出为全局的，之后使用 selenium 请求此文件地址，并返回浏览器对象，再通过浏览器对象执行 js 代码调用方法即可

2. VM2 框架补环境

vm2框架：VM2 是一个基于 Node.js 的 JavaScript 沙箱库，用于在一个安全的环境中执行 JavaScript 代码，提供了一种隔离 JavaScript 代码执行的方式，可以防止恶意代码对系统造成损害。

作用：vm2 提供了一个干净独立的沙盒环境，提供测试，打造了一个真实的浏览器环境，可以用于逆向环境

框架配置：

  1. npm install vm2
  2. arch -arm64 brew install pkg-config cairo pango jpeg giflib librsvg
  3. npm install canvas
  • vm2框架使用教程：
    1. run/run.js：需要补环境的代码，即逆向代码（其中需要导出关键方法为 window 全局方法）
    2. run/lastDeal.js：调用 window 全局方法
    3. config/config.js：全局对象的某些基础属性配置（location.href、navigator.userAgent、cookie等）
    4. /main.js：主函数调用，保证逆向代码 run.js 文件路径配置正确
    5. tools/printLog.js（可选）：日志文件路径 filePath 配置（可以在 run/log.txt 查看吐的环境）

三、Python 与 JS 互调

1. pyexecjs 直接执行 js

简介：PyExecJS 是一个用于在 Python 中执行 js 代码的库。它的设计目标是在不需要绑定到特定 JavaScript 引擎的情况下提供灵活性，直接利用系统中已经安装的 JavaScript 运行时，如 Node.js、Rhino 等（pip install PyExecJS）

  • execjs.get()：

    • 作用：用于获取系统中可用的 JavaScript 运行时环境

    • 返回值：返回一个 Runtime 对象，可以使用这个运行时环境对象来编译和执行 JavaScript 代码

  • Runtime.compile(source)：

    • 作用：用于编译 JavaScript 代码，接受字符串参数 source 表示要执行的 JavaScript 代码
    • 返回值：返回一个 Context 对象，表示一个 JavaScript 上下文

  • Context.call(name, *args)：

    • 作用：用于在 JavaScript 上下文中调用函数，接受一个函数名 name 和任意数量的参数 *args
    • 返回值：返回被调用函数的返回值

  • Context.eval(source)：用于在 JavaScript 上下文中执行一段 JavaScript 代码，并返回执行结果

  • 链式调用：result = execjs.compile(open('./ceshi.js').read()).call('add', 1, 2)

2. express 启动 node 服务

为什么使用 express：

  • 在浏览器环境中，process 对象是 undefined 的，而 process 是 nodejs 特有的系统级 API，用于访问与操作系统和当前运行环境相关的信息，在补环境时，如果需要检测需要定义为 undefined，此时如果使用 execjs 则会报错 execjs._exceptions

简介：Express.js 是一个流行的、基于 Node.js 的 Web 应用程序框架，简化了在 Node.js 中构建 Web 应用程序和 API 的过程

应用场景：客户只需要一个API接口，用于返回数据，而不需要项目的源代码

安装：npm install express -S（将Express安装为项目的生产环境依赖，即安装在项目这个局部）

  • 路由：Express通过路由将HTTP请求与特定的处理函数关联起来，可以根据不同的URL路径将不同的请求映射到相应的处理函数

四、字体反爬专题

字体反爬工作原理：

  1. 字体文件替代字符映射：网页使用自定义字体文件，将常规的字符映射到不同的字形。例如，字母”A”在字体文件中可能被映射为一个特定的符号，而不是标准的字母形状。这样，即使在HTML源代码中看到了字母”A”，实际显示的可能是另一种字符。
  2. 动态生成字体文件：有些网站会在每次加载页面时动态生成字体文件，使得每次访问页面时，字体文件都会有所变化。这增加了破解字体映射的难度，因为爬虫需要每次重新分析字体文件。

1. 字体文件格式

  • 常见字体文件格式：
    • TTF：一种矢量字体格式，支持轮廓字形和平滑的曲线。用于 windows 操作系统、桌面应用程序和网页
    • OTF：一种可扩展的字体格式，TTF的改进版，支持 unicode 字符集。常用于印刷、出版和网页设计
    • FON：通常用于存储位图字体，不支持矢量图形，将每个字符表示为像素图。用于早期的 windows 系统
    • TTC：是 TrueType 字体的集合格式，包含多个字体文件。常用于节省空间，以便同时提供多个字体样式
    • SHX：一种 AutoCAD 使用的字体文件格式。常用于图形设计和 CAD 软件中，支持矢量图形和线框字形
    • EOT：由 Microsoft 开发的一种字体格式，支持字体子集化和压缩。主要用于早期浏览器中网页字体嵌入
    • WOFF：为 Web 优化的字体格式，是 TTF 和 OTF 的包装版，支持压缩和元数据。主要用于网页嵌入字体

2. 字体文件处理

定位：一般通过 @font-face 定位字体映射文件的下载地址

解析：通过**在线字体解析**，将文件直接拖入解析，查看字体映射关系

分析：根据在线字体解析出的映射关系，分析网页源代码字体与文件映射关系的一一对应

  • 反反爬策略：
    1. 检查网页控件的字体名称，即 class 样式名称（对应 @font-face 中 font-family 的值）
    2. 通过首页源码 @font-face 搜索、Network-Font面板、全局搜索 class 样式进行调试分析获取字体地址
    3. 通过在线字体解析网站查看字体的映射关系，配合网页源代码分析
    4. 通过页面刷新、下载查看字体，判断字体文件是否动态生成
    5. 使用依赖库 fonttools 解析字体，根据元素内容生成新的键值对，对响应结果根据键值对进行内容替换
  • 工具库 fonttools 的使用：
    • cmap - 字符到字形映射：建立字符码和字形索引之间的映射，将文本字符映射到字体文件中的字形
    • glyf - 字形数据：包含了字体中每个字形的轮廓和其他相关数据。轮廓数据描述了字体中每个字符的形状
